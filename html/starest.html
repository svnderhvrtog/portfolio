Concept

This project was my final exam for my front-end development course. It required building a mobile application using a JavaScript framework, and I chose Vue.js. To enhance touch interaction, I used Vuetify, while Bootstrap provided the components, and Pinia handled state management.

The project required the following features:

  • Submitting reports
  • Viewing executed reports through a custom-built API
  • Accessing files in a dedicated knowledge base
  • Customizing system preferences and profile settings
  • User authentication using state management
  • Seamless navigation via routing

Build

I developed this project through multiple iterations, continuously refining it to achieve the final product. The application is built with several asynchronously loaded components, making it a Single Page Application (SPA). This improves the user experience by eliminating page reloads, ensuring seamless navigation and faster interactions. Let me highlight some of the interesting features of the code!

I began by developing several foundational components and then integrated them using routing to establish logical connections, including the bottom navigation bar. Later, I added the meta functionality to ensure that users can only access pages if they are authenticated.

Improve

Submitting reports was a feature that broke during implementation on my live website. However, I managed to get it working properly by using a locally hosted API instead of the dummy API. The dummy API can only read and store data, but it doesn't save new submissions. In a real use case, inspectors would be able to select options like "Deferred Maintenance," fill out the form, upload photos, and more.

Once the assignment was complete, I enhanced the design further, incorporated new elements of the corporate identity, and fixed some bugs. Feel free to test the mobile application yourself! I've provided the login details so you can access all the pages.
