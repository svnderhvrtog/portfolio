Concept

Oefenen, oefenen, oefenen: dat is het advies dat veel mensen geven om coderen onder de knie te krijgen. Dus volgde ik het op. Dit project is een van mijn pogingen om een ​​variant te creëren van een al “overbekend” concept, Tinder. Door dat te doen, kon ik mijn focus verleggen van design naar de codeer aspecten.

Ik wilde een aantal essentiële onderdelen integreren, namelijk:

  • De mogelijkheid om te 'liken' en 'disliken'
  • De mogelijkheid om te chatten
  • Een overzicht van de gebruikers-geschiedenis geven
  • De optie om de web applicatie af te sluiten en later verder te gaan

Design

In plaats van een compleet design system vanaf nul op te bouwen, deed ik inspiratie op online en creëerde ik schermen die een versie van Tinder vertegenwoordigen, met de essentiële onderdelen die ik wilde. Ik streefde naar een minimalistisch design waarbij de foto's centraal staan, waardoor ze het opvallendste element worden. Om de interface clean te houden, gebruikte ik intuïtieve iconen en een eenvoudige lay-out.

Build

Het was een logische keuze om een ​​API te gebruiken voor dit project. Het stelt de gebruiker in staat om 'oneindig' te swipen, terwijl het mij de moeite bespaart om een eigen cat database te maken en te onderhouden. Bovendien heb ik een sterke voorkeur voor asynchrone webpagina's, zoals ik in mijn andere projecten ook heb gedaan, dus ik wilde die functionaliteit hier ook integreren. Gezien de onderdelen die ik wilde integreren, was het essentieel om dit project te bouwen met een focus op DOM-manipulatie.

Terwijl ik de DOM dynamisch manipuleerde, moest ik ervoor zorgen dat chats en lijsten verschenen als reactie op de interacties van de gebruiker. Dit betekende dat ik continu data moest pushen en renderen, wat een behoorlijke uitdaging bleek om te implementeren.

Improve

De 'CatChat' was het gebied dat de meeste verbetering behoefde in de loop van de tijd, dit vanwege de neiging tot verschillende fouten. Bijvoorbeeld, de typ indicator verstoorde de lay-out, waardoor alle inhoud niet meer in de juiste volgorde stond. Berichten verdwenen zodra ze het einde van de container bereikten, of de gebruiker moest voor elk nieuw bericht naar beneden scrollen. Ik heb deze problemen aangepakt met een combinatie van CSS- en JavaScript-aanpassingen, die nu naadloos werken.

Code

-

                    
// Functie die echte kat interactie simuleert
function catReplyFunc() {
    isTyping.style.visibility = 'visible';
    setTimeout(() => {
        const chatCatMessage = document.createElement('div');
        chatCatMessage.classList.add('cat-message');
        const chatCatText = document.createTextNode(catReplies());
        chatCatMessage.appendChild(chatCatText);
        chatFrame.appendChild(chatCatMessage);
        isTyping.style.visibility = 'hidden';
        scrollToBottom();
    }, 2000);
};

// Functie die automatisch naar beneden scrollt
function scrollToBottom() {
    const chatFrame = document.getElementById('chat-frame');
    chatFrame.scrollTop = chatFrame.scrollHeight;
};
